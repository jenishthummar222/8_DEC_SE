            -->  Keywords  :-
                    ~>  Special  predefined word

                        ~> alter table <table_nm> <alter_command _nm> filed_nm  datatype.

                        ~> alter table <table-nm> modify <col_nm> <dataType>;

            --> Drop :-

                    Syntax:-
                        ~>  alter table <table_nm> <drop> filed_nm;

                        ~> alter table student drop dummy;

    Theory Questions: 

    -------------------------------------------------------------------------         
 
        1. What is the function of the DROP command in SQL? 

            --> Drop is use for delete database or table form the systems.
            --> Using Drop command you can drop table or database .
            --> Drop use with alter command you just drop column.

            Syntax:-
                Drop tabel <Table_nm>;

        !!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!

        2. What are the implications of dropping a table from a database? 

            --> Dropping a table in a database using DROP TABLE table_name;

            --> 1. Permanent Deletion of Data :-

                    ~> All data inside the table is completely deleted.

            --> 2. Table Structure is Removed :-

                    ~> The table definition (columns, data types, constraints) is also deleted.
            
            --> 3. Loss of Relationships :-

                    ~> If the table is connected to other tables using foreign keys, those relationships will be affected.

10] Transaction Control Language (TCL)     

    Theory Questions: 

    -------------------------------------------------------------------------         
 
         
        1. What is the purpose of the COMMIT and ROLLBACK commands in SQL?

            --> COMMIT :-
                    ~> it is permenet save all changes in databsae.
                    ~> once you do commit you can't be do rollback.

                    E.g :-
                        ~> start transaction;

                        ~> UPDATE student
                            SET name = 'Ram'
                            WHERE id = 1;

                        ~> COMMIT;

            --> ROLLBACK :-
                    ~> it is use for roll back to start transation.
                    ~> it is work like undo on changes.
                
                    E.g :-
                        ~> start transaction;

                        ~> UPDATE student
                           SET name = 'Shyam'
                           WHERE id = 1;

                        ~> ROLLBACK;
                    
        !!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
        
        2. Explain how transactions are managed in SQL databases. 

            --> A transaction is a group of SQL operations executed as a single unit of work.
             
            --> it's have 3 part 

                1} RollBack
                2} CheckPoint 
                3} Commit 

            --> Transaction Flow:-

                    ~> START TRANSACTION;

                    ~> SQL operations

                        UPDATE account SET balance = balance - 500 WHERE id = 1;
                        UPDATE account SET balance = balance + 500 WHERE id = 2;

                    ~> COMMIT;

            --> If everything is correct → COMMIT
            --> If error occurs → ROLLBACK

            SavePoint :-
                --> we can create check point and we can jump on specific checkpoint.

                E.g :
                    ~> SAVEPOINT sp1;
                    
                E.x:-
                    ~> START TRANSACTION;

                    ~> SQL operations

                        UPDATE account SET balance = balance - 500 WHERE id = 1;

                        SAVEPOINT sp1;

                        UPDATE account SET balance = balance + 500 WHERE id = 2;

                        SAVEPOINT sp2;

                    ~> ROLLBACK to sp1;

11] SQL Joins   

    Theory Questions: 

    -------------------------------------------------------------------------         
 
         
        1. Explain the concept of JOIN in SQL. What is the difference between INNER JOIN, LEFT JOIN, 
           RIGHT JOIN, and FULL OUTER JOIN? 

            --> JOIN :-

                ~> when we wants to join 2 or more tables with using of same matching value its called joining 

                ~> There is no need to apply Foreign Ky for joining.

            1. INNER JOIN :-                
               
                    ~> Returns only matching records from both tables
                
                Result: ->  Non-matching rows are excluded

                Example Result: -> Common data only

            2. LEFT JOIN (LEFT OUTER JOIN)
            
                    ~> Returns all records from left table
                    ~> matching records from right table
                    ~> If no match → NULL
                Example Result: ->   All Left + Matching Right

            3. RIGHT JOIN (RIGHT OUTER JOIN)
            
                    ~> Returns all records from right table
                    ~> matching records from left table
                    ~> If no match → NULL
                Example Result: -> All Right + Matching Left

            4. FULL OUTER JOIN
            
                    ~> Returns all records from both tables
                    ~> If no match → NULL on either side

                Example Result: -> All Left + All Right (including unmatched)
                
        !!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
        
        2. How are joins used to combine data from multiple tables ?  

            --> JOINs are used to link tables using a common column (usually a Primary Key ↔ Foreign Key) 
               so you can fetch related data in one query.

            Syntax :-
                SELECT s.name, c.course_name, o.order_id
                FROM student s
                JOIN course c ON s.course_id = c.course_id
                JOIN orders o ON s.id = o.student_id;
